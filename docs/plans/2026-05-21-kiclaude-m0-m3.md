## 0. Plan-level commitments

These are the dev-practice defaults this plan operates under. They were chosen — not punted — because the user is in auto mode. To change one, edit this section and the plan is updated; do not litigate them inside a task.

| # | Decision | Why |
|---|---|---|
| C1 | **TDD where mechanical, test-after where exploratory.** Parsers, KCIR validators, geometry, DRC kernel, impedance solver, schema migrations: tests first (failing → impl → green). UI gestures, layout polish, debug viz: test-after with explicit assertions on outcomes. | Tests-first pays off where invariants are clear (round-trip, math). It taxes you where the design is being discovered. Honest about the split. |
| C2 | **One branch per task.** Branch name = task id slugified: `m0-r-01-cargo-workspace`. Squash-merge. Conventional Commits. | Plan ids map 1:1 to git history; review surface stays small. |
| C3 | **Two-stage review per task.** Self-review then `lore:code-reviewer` subagent before merge. Reviewer findings tracked as sub-tasks of the original task, not new tasks. | Matches the spec's quality posture without inventing process. |
| C4 | **Plan is executable via `/lore:execute`.** Each task is a checkbox; `done when` is testable; deps are explicit. | Lets the planning artifact drive the work without a separate tracker. |
| C5 | **Spec is the contract.** When a task and the spec disagree, the spec wins; the task is wrong and must be edited. To change the spec, write a SPEC revision, not a one-line plan edit. | Avoids quietly diverging plans. |
| C6 | **No stubs at task close.** A task is closed only when its `done when` is fully met; "stub for now" is not acceptable. Tasks that need a follow-up record the follow-up as a separate task in this file, not as a TODO comment in code. | First principles of this project; reinforces global rules. |
| C7 | **Golden-file CI is non-negotiable.** Every PR touching `crates/ki` or `services/kiserver` runs round-trip CI against all `examples/**`. Failure blocks merge. | Spec §13 gate #4 + first principle #1. |
| C8 | **Quality gates from SPEC §13 are CI lints, not aspirational.** Cargo fmt/clippy/test, pnpm typecheck/lint/test, uv ruff/mypy/pytest, golden-file, KCIR-version-bump, license audit, OTel-span contract, axe-core. Every PR. | Same. |

**Working calendar:** 5 working days/week. Estimated durations below are "ideal engineering days" with no meetings. Real wall-clock will be longer; budget +25% for actual schedule.

**Schedule honesty:** the M0–M3 spec target sum is 34 weeks. This plan sums to **~38 weeks of ideal engineering** (so ~48 weeks wall-clock at +25%). M2 and M3 overshoot the spec's per-milestone targets; the bottlenecks are the Rust walk-around router (M2) and the push-and-shove port (M3). Flag at the bottom of this plan — see §6.

## 2. Task notation

```
- [ ] **M{N}-{L}-{NN}** — Short title (Xd)
    Files: path/a, path/b
    Done when: testable success criterion
    Deps: M{N}-{L}-{NN}, …
    Spec refs: FR-NNN, §X.Y
```

Layer codes: **R** = Rust (`crates/`), **P** = Python (`services/`), **T** = TypeScript (`client/`, `services/server`, `packages/`), **C** = `.claude/` assets, **Q** = quality / CI / tests.

Durations are ideal-engineering days. `(Xd)` where X ∈ {0.5, 1, 1.5, 2, 2.5, 3, …}.

## 9. Execution handoff

This plan is consumable by `/lore:execute`. Two execution shapes:

1. **Sequential inline.** Run `/lore:execute` and the agent will walk the tasks top-to-bottom, opening one PR per task, blocking on `done when` verification.
2. **Wave-based parallel.** Tasks within a single layer-group (e.g., M0-R-01..09) are inherently sequential due to deps; tasks across layer-groups in the SAME milestone are largely parallelizable. M0 has three natural waves:
    - **Wave 1:** M0-R-01 + M0-P-01 + M0-T-07 (no deps).
    - **Wave 2:** everything that depends only on the wave-1 trio.
    - **Wave 3:** integration + CI tasks.
   See `/lore:dispatching-parallel-agents` for the dispatch pattern.

**Recommended start:** Sequential for M0 (the plumbing has hidden coupling that benefits from one mind); switch to wave-based at M1.
